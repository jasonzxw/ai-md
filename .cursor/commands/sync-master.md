# sync-master

## 概述
将远程 master 分支的最新代码合并到当前分支，保持当前分支与主分支同步。

## 执行步骤

请依次执行以下 Git 操作，并在每一步后检查结果：

### 第一步：查看当前状态
```bash
git status
```
确认当前所在分支，以及工作区是否有未提交的改动。

### 第二步：暂存未提交改动（如有）
如果工作区存在未提交的改动，先暂存：
```bash
git stash
```

### 第三步：拉取远程 master 最新代码
```bash
git fetch origin master
```

### 第四步：对比 master 与当前分支的差异
在合并之前，先查看 master 相比当前分支新增了哪些提交：
```bash
git log HEAD..origin/master --oneline
```
逐条列出每条提交的 hash 和提交信息，并用一句话总结 master 本次带来了哪些功能或改动。如果没有新提交，说明 master 与当前分支已一致，可跳过合并步骤。

### 第五步：合并 master 到当前分支
```bash
git merge origin/master
```
如果合并过程中出现冲突，列出所有冲突文件及冲突位置，协助逐一分析。
若冲突过于复杂、超出预期，**不要强行解决**，立即执行以下命令撤回到合并前的干净状态：
```bash
git merge --abort
```
执行后用 `git status` 确认工作区已恢复，并告知用户已成功撤回，建议下次合并前先逐文件评估冲突范围。

### 第六步：恢复暂存的改动（如第二步执行了 stash）
```bash
git stash pop
```


## 注意事项
- 如果合并过程中出现**冲突**，请列出冲突文件，并逐一说明冲突的位置和内容，辅助解决冲突。
- 若冲突复杂超出预期，执行 `git merge --abort` 撤回合并，并用 `git status` 确认恢复成功。
- 合并完成后，输出当前分支的最新 git log（最近 5 条），确认同步结果。
- 整个过程中若有任何报错，立即停止并报告错误信息，不要自行假设继续执行。
