# git-help

## 概述
根据用户的描述，在 Git 指令手册中查找匹配的指令，给出详细用法和示例。

## 输入
用户会用自然语言描述想要执行的 Git 操作，例如：
- "怎么撤销上一次提交"
- "如何删除远程分支"
- "怎么把多个提交合并成一个"
- "查看某个文件是谁改的"

## 执行步骤

### 第一步：读取 Git 指令手册
读取 `docs/Git指令完全手册.md` 文件内容，作为查询知识库。

### 第二步：理解用户意图
分析用户的描述，识别其核心需求：
- 用户想完成什么操作？
- 涉及哪个 Git 概念？（分支、提交、远程、暂存、回退、合并等）
- 是否有特殊条件？（已推送、公共分支、需保留历史等）

### 第三步：匹配并输出指令
从手册中找到最匹配的指令，按以下格式回复：

```
## 🔍 查询：<用户的描述>

### 推荐指令

<给出最匹配的命令，附带注释说明>

### 含义说明

<用通俗语言解释这条命令做了什么，以及每个参数的作用>

### 使用示例

<给出 1~3 个贴合实际场景的示例>

### ⚠️ 注意事项

<如果该操作有风险或需要注意的地方，在此说明>
```

### 第四步：补充相关指令（可选）
如果有相关的替代方案或组合用法，一并给出，帮助用户全面了解。例如：
- 用户问"怎么撤销提交"→ 同时给出 `reset`、`revert`、`reflog` 三种方案及各自适用场景
- 用户问"怎么合并分支"→ 同时给出 `merge`、`rebase`、`squash merge` 的区别

## 回复规范
- 使用中文回复
- 指令用代码块包裹，便于复制
- 如果用户描述模糊，先列出可能的几种理解，让用户确认
- 危险操作（如 `--force`、`--hard`）必须标注警告
