# pdf-to-markdown

## 概述
将需求文档（PDF）转换为 Markdown 文件，方便在项目中阅读和引用。调用 `scripts/pdf_to_markdown.py` 脚本完成转换。

## 输入要求
用户需提供以下信息：
- **PDF 文件路径**（必填）：需求文档的路径，支持单个文件或整个文件夹
- **输出目录**（可选）：默认输出到 `docs/` 目录
- **页码范围**（可选）：从 0 开始，如 `0-5`、`1,3,5`、`2-`
- **是否提取图片**（可选）：默认提取，用户可指定不提取

如果用户没有指定 PDF 路径，提示用户提供。

---

## 执行步骤

### 第一步：确认输入参数

从用户输入中提取以下参数：

| 参数 | 来源 | 默认值 |
|------|------|--------|
| PDF 路径 | 用户提供 | 无（必填） |
| 输出目录 | 用户提供或默认 | `docs/` |
| 页码范围 | 用户提供 | 全部页码 |
| 提取图片 | 用户提供 | 是 |

如果用户提供的是相对路径，基于项目根目录解析。

### 第二步：检查 Python 环境

执行以下命令确认依赖已安装：
```bash
pip install -r scripts/requirements.txt
```

如果安装失败，提示用户检查 Python 环境。

### 第三步：执行转换

根据参数组装命令并执行：

**单个文件转换：**
```bash
python scripts/pdf_to_markdown.py -i <PDF路径> -o <输出目录>
```

**整个文件夹批量转换：**
```bash
python scripts/pdf_to_markdown.py -d <文件夹路径> -o <输出目录>
```

**附加可选参数：**
- 指定页码：追加 `--pages <页码范围>`
- 不提取图片：追加 `--no-images`

### 第四步：输出结果

转换完成后，向用户展示结果摘要：
```
## ✅ 需求文档转换完成

**输入**：<PDF 文件路径>
**输出**：<生成的 Markdown 文件路径>
**图片**：已提取到 <图片目录> / 未提取

### 输出文件
- `docs/xxx.md` — 需求文档 Markdown
- `docs/xxx_images/` — 提取的图片（如有）

### 建议下一步
1. 阅读转换后的 Markdown 文档，确认内容完整
2. 执行 `/generate-tech-design` 基于需求文档生成技术方案
```

---

## 示例用法

```
/pdf-to-markdown
文件在 D:/docs/需求文档v2.pdf
```

```
/pdf-to-markdown
把 D:/docs/ 下所有 PDF 都转换一下，不需要提取图片
```

```
/pdf-to-markdown
转换 D:/docs/report.pdf 的前 5 页，输出到 docs/ 目录
```

## 注意事项
- 支持的文件格式仅限 PDF
- 大文件转换可能需要较长时间，请耐心等待
- 如果 PDF 中包含扫描图片（非文字 PDF），提取效果可能受限
- 转换后建议人工检查格式，特别是表格和图片部分
- 所有回复使用中文
